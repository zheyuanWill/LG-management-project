## 1. 功能概述

### 这个功能是做什么的？

工作流编排是一个**可视化的业务流程设计器**，用于：

- **设计**修船/供应链项目的标准化执行流程（模板）
- **跟踪**每张订单的实际执行进度
- **自动化**条件判断、并行任务分发、审批节点等

### 解决什么问题？

| 痛点 | 工作流如何解决 |
|------|---------------|
| 不同项目类型流程不同，容易遗漏步骤 | 按项目类型设计标准模板，确保每个步骤都不遗漏 |
| 项目执行进度不透明 | 工作流跟踪器实时显示每个节点的状态 |
| 流程中需要人工判断走哪个分支 | 条件节点根据订单金额、类型等自动路由 |
| 多个任务可以同时进行但经常串行 | 并行网关支持多任务同时推进 |
| 谁在什么时候做了什么操作没有记录 | 审计日志完整记录每次节点状态变更 |

### 谁可以使用？

- **OWNER（老板）**和 **PM（项目经理）** 可以进入工作流编排页面设计模板
- 所有有跟单权限的用户可以在订单详情中查看和推进工作流

## 2. 核心概念

### 2.1 模板 (Template)

模板是一个流程的**设计蓝图**，包含节点和连线的定义。模板可以复用——一个模板可以应用到多张订单上。

```
模板 = 节点(Nodes) + 连线(Edges) + 元信息(名称、项目类型、版本)
```

### 2.2 实例 (Instance)

实例是模板在某张订单上的**运行副本**。创建实例后，系统会自动初始化所有节点状态，并从「开始」节点出发推进流程。

```
实例 = 模板快照 + 订单绑定 + 各节点运行状态 + 审计日志
```

### 2.3 节点 (Node)

节点是流程中的一个步骤。每个节点有类型（任务、审批、条件等）和状态（待办、进行中、已完成等）。

### 2.4 连线 (Edge)

连线表示节点之间的执行顺序。条件节点的连线会标注「是」或「否」。

## 3. 快速上手

### 第一步：进入编排页面

侧边栏 → **工作流编排**（仅 OWNER 和 PM 可见）

### 第二步：拖拽节点到画布

从左侧面板拖拽节点到中央画布：

1. 先拖一个「**开始**」节点
2. 拖入业务节点（任务、审批等）
3. 最后拖一个「**结束**」节点

### 第三步：连接节点

从一个节点右侧的**输出端口**（小圆点）拖向另一个节点左侧的**输入端口**，即可建立连线。

### 第四步：配置节点

- **单击**节点：右侧显示配置面板，可修改属性
- **双击**节点：弹出编辑对话框

### 第五步：验证 & 保存

1. 点击「**验证**」按钮检查流程是否完整
2. 点击「**保存模板**」，输入名称和描述
3. 模板即可在订单中使用

## 4. 节点类型详解

### 4.1 基础节点

#### 开始节点 (Start)

- **用途**：流程的入口，每个流程**有且仅有一个**
- **特点**：只有输出端口，无输入端口
- **创建实例时**：自动标记为「已完成」并推进到下一个节点

#### 结束节点 (End)

- **用途**：流程的出口，每个流程**有且仅有一个**
- **特点**：只有输入端口，无输出端口
- **当到达时**：整个工作流实例标记为「已完成」

#### 任务节点 (Task)

- **用途**：需要执行的具体工作任务
- **可配置属性**：
  - `标题` — 任务名称
  - `描述` — 详细说明
  - `预计天数` — 计划完成天数
  - `负责人` — 任务执行人
- **典型场景**：备料、检验、发货、报关等

#### 审批节点 (Approval)

- **用途**：需要审批确认的环节
- **可配置属性**：同任务节点
- **典型场景**：报价审批、采购审批、合同审批

#### 通知节点 (Notification)

- **用途**：在流程推进时发送通知
- **可配置属性**：
  - 通知渠道：`邮件`、`短信`、`微信`（多选）
- **典型场景**：订单确认通知、发货通知、完工通知

#### 里程碑节点 (Milestone)

- **用途**：标记流程中的关键节点
- **可配置属性**：
  - `标题`、`描述`、`预计天数`
- **典型场景**：合同签署、货物到港、项目验收

### 4.2 控制流节点

#### 条件分支 (Condition)

- **用途**：根据条件自动选择执行分支
- **输出端口**：两个 —「✓ 是」和「✗ 否」
- **可配置属性**：
  - `条件表达式` — 支持的变量和语法见[第9节](#9-条件表达式语法)
- **运行时行为**：
  - 自动获取订单上下文（金额、类型等）
  - 计算表达式结果
  - `true` → 走「是」分支，`false` → 走「否」分支
  - 未被选中的分支所有节点标记为「已跳过」

**示例**：

```
条件: amount > 100000
├── 是 → 总经理审批 → ...
└── 否 → 部门经理审批 → ...
```

#### 并行网关 (Parallel Gateway)

- **用途**：让多个任务同时执行，或等待多个任务都完成后再继续
- **两种模式**：
  - **Fork（分发）**：同时激活所有下游节点
  - **Join（汇聚）**：等待所有上游节点完成后才继续
- **典型用法**：Fork 和 Join 成对使用

**示例**：

```
Fork 并行网关
├── 采购配件 (任务)
├── 安排技师 (任务)
└── 准备工具 (任务)
Join 并行网关 → 开始施工
```

### 4.3 高级节点

#### 定时器 (Timer)

- **用途**：在流程中设置延迟等待
- **可配置属性**：
  - `延迟小时数`
  - `延迟分钟数`
- **典型场景**：等待材料固化、等待审批窗口期
- **注意**：当前为记录状态，未接入实际定时调度

#### 子流程 (Subprocess)

- **用途**：在当前流程中嵌套引用另一个工作流模板
- **可配置属性**：
  - `引用模板` — 从已有模板列表中选择
- **典型场景**：复杂项目拆分为主流程 + 子流程
- **注意**：当前为记录状态，未实现自动创建子实例

## 5. 画布操作

### 5.1 基本操作

| 操作 | 方式 |
|------|------|
| 添加节点 | 从左侧面板拖拽到画布 |
| 连接节点 | 从输出端口拖拽到输入端口 |
| 选中节点 | 单击节点 |
| 编辑节点 | 双击节点，或选中后在右侧面板修改 |
| 删除节点 | 选中后按 `Delete` 键，或节点菜单中选择删除 |
| 复制节点 | 选中后按 `Ctrl+D`，或节点菜单中选择复制 |
| 平移画布 | 拖拽空白区域 |
| 缩放画布 | 鼠标滚轮 |

### 5.2 高级操作

| 操作 | 说明 |
|------|------|
| 自动布局 | 按拓扑顺序自动排列节点位置（`Ctrl+L`） |
| 撤销/重做 | 最多支持 50 步历史记录 |
| 验证流程 | 检查流程完整性，高亮有问题的节点 |
| 清空画布 | 删除所有节点和连线，需确认 |

### 5.3 节点菜单

右键或点击节点右上角的菜单按钮：

- **编辑** — 打开编辑对话框
- **复制** — 在偏移位置创建副本
- **删除** — 删除节点及其所有连线

## 6. 模板管理

### 6.1 保存模板

1. 设计好流程后，点击左侧面板的「**保存模板**」按钮（或 `Ctrl+S`）
2. 填写：
   - **模板名称**（必填）
   - **描述**（可选）
   - **项目类型**（自动填充当前选择的类型）
3. 点击「保存」

> 如果已加载某个模板并修改，保存时会更新该模板并自动递增版本号。

### 6.2 加载模板

1. 点击「**加载模板**」按钮
2. 在模板列表中浏览已有模板
3. 点击模板名称加载到画布
4. 可以修改后另存或更新

### 6.3 删除模板

在模板列表中，点击模板右侧的删除按钮，确认后删除。

### 6.4 模板属性

| 属性 | 说明 |
|------|------|
| 名称 | 模板名称，如「技术服务标准流程」 |
| 描述 | 模板用途说明 |
| 项目类型 | 关联的项目类型（技术服务、监理等） |
| 版本号 | 每次更新自动递增 |
| 节点数 / 连线数 | 流程复杂度指标 |
| 创建者 | 谁创建的模板 |
| 是否激活 | 停用的模板不会出现在可选列表中 |

## 7. 图验证规则

点击「验证」按钮后，系统会检查以下规则：

### 错误（必须修复）

| 规则 | 说明 |
|------|------|
| 必须有且仅有一个开始节点 | 流程需要明确的入口 |
| 必须有且仅有一个结束节点 | 流程需要明确的出口 |
| 所有节点必须从开始节点可达 | 不能有孤立的节点组 |
| 条件节点必须有「是」和「否」两个分支 | 否则条件判断无法路由 |
| 并行 Fork 网关至少有 2 个输出 | Fork 少于 2 个输出没有意义 |
| 并行 Join 网关至少有 2 个输入 | Join 少于 2 个输入没有意义 |

### 警告（建议修复）

| 规则 | 说明 |
|------|------|
| 存在未连接的节点 | 节点没有输入或输出连线 |
| 结束节点不可达 | 某些路径无法走到结束 |
| 存在循环 | 流程中有环路，可能导致无限循环 |
| 定时器未配置延迟时间 | 定时器节点没有设置延迟 |
| 子流程未选择模板 | 子流程节点没有关联模板 |

## 8. 工作流实例与跟踪

### 8.1 创建实例

在**订单详情**页面（订单状态非「草稿」时），底部会显示工作流跟踪面板：

1. 如果该订单还没有工作流实例，会显示「暂无工作流实例」
2. 点击「创建工作流实例」选择模板
3. 系统自动：
   - 复制模板定义到实例
   - 获取订单上下文（金额、类型、客户等）
   - 初始化所有节点为「待办」状态
   - 自动完成「开始」节点
   - 根据条件自动推进可自动推进的节点

### 8.2 节点状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| PENDING（待办） | 灰色 | 尚未开始 |
| RUNNING（进行中） | 蓝色/动画 | 当前正在执行 |
| COMPLETED（已完成） | 绿色 | 已成功完成 |
| SKIPPED（已跳过） | 灰色/斜线 | 被条件分支跳过 |
| FAILED（失败） | 红色 | 执行失败 |
| WAITING（等待中） | 橙色 | 等待前置节点（Join 网关） |

### 8.3 推进节点

在跟踪面板中，当前为「进行中」状态的节点会显示操作按钮：

- **完成** — 将节点标记为已完成，自动推进到下一个节点
- **跳过** — 将节点标记为已跳过，同样推进到下一个节点

推进时可以添加备注说明。

### 8.4 自动推进

以下节点类型在前置节点完成后会**自动推进**，无需手动操作：

- **开始节点** — 实例创建时自动完成
- **条件节点** — 自动评估表达式并路由
- **并行 Fork** — 自动激活所有下游
- **并行 Join** — 所有上游完成后自动完成
- **结束节点** — 到达时自动完成整个实例

### 8.5 审计日志

每次节点状态变更都会记录审计日志，包含：

- 时间戳
- 操作类型（实例启动、节点推进、条件评估等）
- 节点 ID
- 状态变更（从 → 到）
- 操作人
- 详细信息

在跟踪面板底部的「审计日志」区域可以展开查看。

## 9. 条件表达式语法

条件节点支持使用以下变量和运算符编写表达式：

### 可用变量

| 变量名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `amount` | 数字 | 订单总金额 | `150000` |
| `project_type` | 字符串 | 项目类型 | `"TECHNICAL_SERVICE"` |
| `status` | 字符串 | 订单状态 | `"IN_PROGRESS"` |
| `currency` | 字符串 | 货币类型 | `"CNY"` |
| `customer_name` | 字符串 | 客户名称 | `"XX船务公司"` |
| `days_elapsed` | 数字 | 已耗天数 | `15` |

### 运算符

| 类型 | 运算符 |
|------|--------|
| 比较 | `>`, `<`, `>=`, `<=`, `==`, `!=` |
| 逻辑 | `and`, `or`, `not` |
| 算术 | `+`, `-`, `*` |

### 示例表达式

```python
# 大额订单审批
amount > 100000

# 特定项目类型
project_type == "TECHNICAL_SERVICE"

# 组合条件
amount > 50000 and project_type == "SUPERVISION"

# 超期判断
days_elapsed > 30

# 外币订单
currency != "CNY"
```

> **安全说明**：条件表达式使用 AST 解析器执行，不使用 `eval()`，不会执行任意代码。

## 12. 典型流程示例

### 示例 1：技术服务标准流程

```
开始
  ↓
接收需求 (任务, 2天)
  ↓
[条件: amount > 100000]
  ├── 是 → 总经理审批 (审批, 2天)
  └── 否 → 部门经理审批 (审批, 1天)
        ↓ (汇合)
确认报价 (任务, 3天)
  ↓
签订合同 (里程碑, 5天)
  ↓
并行 Fork
  ├── 采购配件 (任务, 7天)
  ├── 安排技师 (任务, 3天)
  └── 准备工具 (任务, 2天)
并行 Join
  ↓
现场施工 (任务, 14天)
  ↓
验收确认 (审批, 3天)
  ↓
发送完工通知 (通知: 邮件+微信)
  ↓
结算 (任务, 5天)
  ↓
结束
```

### 示例 2：采购审批简化流程

```
开始
  ↓
提交采购申请 (任务, 1天)
  ↓
主管审批 (审批, 2天)
  ↓
[条件: amount > 50000]
  ├── 是 → 财务审批 (审批, 2天) → 总经理审批 (审批, 3天)
  └── 否 → (直接下一步)
        ↓ (汇合)
下单采购 (任务, 1天)
  ↓
等待到货 (定时器: 72小时)
  ↓
入库验收 (任务, 1天)
  ↓
结束
```

## 14. 常见问题

### Q: 修改模板后，已创建的实例会受影响吗？

**不会。** 创建实例时会复制模板定义的快照，后续模板修改不影响已有实例。

### Q: 可以同时为一张订单创建多个工作流实例吗？

技术上可以，但建议每张订单只关联一个主流程实例。如需多流程，可使用子流程节点。

### Q: 条件节点的表达式在哪里执行？

在**后端**执行。系统会自动获取订单的上下文信息（金额、类型等），使用安全的 AST 解析器计算结果。

### Q: 并行网关的 Join 节点什么时候完成？

当 Join 节点的**所有前置节点**都达到「已完成」或「已跳过」状态时，Join 节点自动完成。

### Q: 定时器和子流程节点目前能用吗？

当前版本中，定时器和子流程节点会记录状态，但**未接入实际的定时调度和子实例创建**。后续版本会集成 Celery 定时任务。

### Q: 撤销操作有限制吗？

最多支持 **50 步**撤销。画布上每次节点增删、连线变更、属性修改都会创建一个历史快照。

### Q: 如何查看工作流执行进度？

进入**订单详情**页面，当订单状态不是「草稿」时，页面底部会显示工作流跟踪面板，实时展示各节点的执行状态和审计日志。
